Common Indexing Mistakes

One million product-catalog objects, three deliberately wrong schema choices. Run each card to see the failure mode. Fix it from the Weaviate console, then re-run to see the difference.

1. Range query without a rangeable index

price_cents has a filterable inverted bucket but no rangeable one. The range query works - but the engine has to walk the bucket value by value to satisfy BETWEEN 30000 AND 40000. On a 1M-object catalog that lands in the hundreds-of-ms range. Enable indexRangeFilters and the same query gets a sorted bitmap structure to consult, ending in single-digit ms.

price_cents int filterable=true rangeable=false 

products.query.fetch_objects(
    filters=(
        Filter.by_property("price_cents").greater_or_equal(30000)
        & Filter.by_property("price_cents").less_or_equal(40000)
    ),
    limit=10,
    return_properties=["name", "brand", "category", "price_cents", "sku"],
)

Press Run query to execute.

2. File path searched with WORD tokenization

BM25 search for /products/cameras/gopro against a WORD-tokenized path field. The slashes and dots are dropped, so the term matches any path containing the literal word gopro anywhere - including accessory listings, review URLs, and competitor-comparison paths under other brands.

spec_sheet_path text tokenization=word 

products.query.bm25(
    query="/products/cameras/gopro",
    query_properties=["spec_sheet_path"],
    limit=10,
    return_properties=["spec_sheet_path", "category", "brand", "name"],
)

Press Run query to execute.

3. Equality filter on a non-filterable property

Filter category == "Cameras > Action" against a property with no inverted filterable bucket. Weaviate refuses upfront with requires inverted index - the category facet on the storefront stops working entirely.

category text filterable=false tokenization=field 

products.query.fetch_objects(
    filters=Filter.by_property("category").equal("Cameras > Action"),
    limit=10,
    return_properties=["name", "brand", "category", "sku", "price_cents"],
)

Press Run query to execute.

Fix each mistake in the Weaviate console, then re-run the same card. The UI never caches results.
