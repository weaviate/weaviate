Common Indexing Mistakes

One million product-catalog objects, three deliberately wrong schema choices. Run each card to see the failure mode. Fix it from the Weaviate console, then re-run to see the difference.

1. Range query with no rangeable index

Range filters on price_cents have no rangeable inverted index and no filterable index, so Weaviate refuses the query outright with requires inverted index. From the catalog operator's POV, the storefront just broke.

price_cents int filterable=false rangeable=false 

import weaviate
from weaviate.classes.query import Filter

client = weaviate.connect_to_local()
products = client.collections.get("IndexingMistakesDemo")

results = products.query.fetch_objects(
    filters=(
        Filter.by_property("price_cents").greater_or_equal(30000)
        & Filter.by_property("price_cents").less_or_equal(40000)
    ),
    limit=10,
    return_properties=["name", "brand", "category", "price_cents", "sku"],
)

Press Run query to execute.

2. File path searched with WORD tokenization

BM25 search for /products/cameras/gopro against a WORD-tokenized path field. The slashes and dots are dropped, so the term matches any path containing the literal word gopro anywhere - for example in unrelated categories.

spec_sheet_path text tokenization=word 

results = products.query.bm25(
    query="/products/cameras/gopro",
    query_properties=["spec_sheet_path"],
    limit=10,
    return_properties=["spec_sheet_path", "category", "brand", "name"],
)

Press Run query to execute.

3. Equality filter on a non-filterable property

Filter category == "Cameras > Action" against a property with no inverted filterable bucket. Same fate as the price card: Weaviate refuses with requires inverted index. The category facet on the storefront stops working.

category text filterable=false tokenization=field 

results = products.query.fetch_objects(
    filters=Filter.by_property("category").equal("Cameras > Action"),
    limit=10,
    return_properties=["name", "brand", "category", "sku", "price_cents"],
)

Press Run query to execute.

Fix each mistake in the Weaviate console, then re-run the same card. The UI never caches results.
